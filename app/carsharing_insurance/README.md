## Описание проекта

Сервис для динамического расчета стоимости страховки в каршеринге на основе машинного обучения. Модель предсказывает вероятность ДТП для каждой поездки, учитывая:

- Профиль водителя (возраст, стаж, семейное положение, платежную дисциплину)
- Характеристики поездки (время, погода, длительность, расстояние)
- Телематические данные (стиль вождения, резкие события)
- Характеристики автомобиля

На основе предсказанной вероятности рассчитывается индивидуальная цена страховки.

## Результаты выполнения лабораторных работ

### Лабораторная работа №1. Постановка задачи и высокоуровневое проектирование

**Бизнес-задача:**  
Сервис решает проблему несправедливого ценообразования на страховку в каршеринге. Аккуратные и рискованные водители платят одинаково, что приводит к оттоку лояльных клиентов и увеличению убытков компании.

**ML-интерпретация:**  
Задача бинарной классификации с предсказанием вероятности ДТП. Входные данные: профиль водителя, характеристики поездки, телематика, данные об автомобиле. Выходные данные: персональный страховой тариф.

**Метрики качества:**
- AUC-ROC (ранжирование рисков)
- Log Loss (калибровка вероятностей)
- Brier Score (качество вероятностных предсказаний)

**Архитектура:**  
Django (веб-слой) → Redis (кэш) → ML модель → PostgreSQL (хранение)

### Лабораторная работа №2. Корректировка архитектуры

**Стратегия валидации:**  
Использована временная кросс-валидация (TimeSeriesSplit) для предотвращения утечки данных. Данные разделены: 80% старых (1-25 ноября) на обучение, 20% новых (26-30 ноября) на тест.

**Предотвращение утечек данных:**
- Исключен признак `accident_probability` (прямой прокси цели)
- Масштабирование и кодирование только на train
- Исключен `driver_id`, используются агрегированные характеристики

**Масштабирование:**  
Ожидаемая нагрузка: до 1 RPS в часы пик. Внедрено кэширование Redis для снижения latency. Допустимое время ответа: <300 мс.

**Обновленная архитектура:**  
Django → Redis → MLflow Registry → ML Model → PostgreSQL. Добавлены компоненты: Redis (кэш), MLflow (версионирование), Prometheus (мониторинг).

### Лабораторная работа №3. Разработка и улучшение ML-модели

**Подготовка данных:**
- Загружены и объединены датасеты: CASCO, телематика, поездки такси, страховые случаи
- Сгенерирован датасет из 48 079 записей за ноябрь 2024 года
- Добавлены новые признаки: `experience_ratio`, `hard_events_per_min`, `night_and_inexperienced`, `style_weather_risk`

**Эксперименты (8 моделей):**

| Модель | AUC-ROC | Log Loss | Brier Score | Latency (ms) |
|--------|---------|----------|-------------|--------------|
| LogisticRegression_Calibrated | 0.7292 | 0.1173 | 0.0264 | 9.26 |
| LogisticRegression | 0.7291 | 0.6382 | 0.2251 | 2.29 |
| RandomForest | 0.7272 | 0.3753 | 0.1142 | 67.30 |
| RandomForest_Optimized | 0.7256 | 0.2373 | 0.0602 | 128.27 |
| XGBoost | 0.6779 | 0.2615 | 0.0732 | 7.91 |
| CatBoost | 0.7049 | 0.4767 | 0.1584 | 5.90 |
| LightGBM | 0.6765 | 0.3284 | 0.0985 | 10.17 |

**Финальная модель:** `LogisticRegression_Calibrated`

**Обоснование выбора:**
- Лучшая калибровка вероятностей (Log Loss 0.1173)
- Высокая скорость инференса (9.26 мс)
- Простая интерпретируемость (линейная модель)
- Маленький размер (~1.2 МБ)

**Анализ ошибок:**
- Основная проблема: ложные срабатывания (модель предсказывает ДТП там, где его нет)
- Рекомендуемый порог классификации: 0.55-0.60
- Слабые места: водители 26-35 лет, агрессивный стиль, плохая погода

### Лабораторная работа №4. Развертывание и интеграция

**Шаг 1. Инференс-модуль**

Реализован класс `InsuranceRiskPredictor` в `ml_service.py`:
- Загрузка модели и препроцессора
- Подготовка признаков (заполнение пропусков)
- Предсказание вероятности ДТП
- Расчет цены с искусственным завышением риска (boost_factor=3.0)

**Шаг 2. Интеграция компонентов**

Все компоненты объединены в единую систему:
- Django ↔ ML сервис (прямой вызов)
- Django ↔ PostgreSQL (хранение логов)
- Django ↔ Redis (кэширование)
- MLflow ↔ ML модель (версионирование)
- Git + DVC (версионирование кода и данных)

**Шаг 3. Мониторинг**

Внедрены метрики Prometheus:
- `http_requests_total` — количество запросов
- `http_request_duration_seconds` — время ответа
- `predictions_total` — количество предсказаний
- `prediction_latency_seconds` — время ML предсказания
- `cache_hits_total` / `cache_misses_total` — эффективность кэша
- `system_cpu_percent` / `system_memory_percent` — ресурсы системы

Эндпоинты:
- `/api/health/` — проверка доступности
- `/api/predict/` — предсказание
- `/api/stats/` — статистика
- `/api/metrics/` — метрики для Prometheus
